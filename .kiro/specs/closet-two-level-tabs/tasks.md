# Implementation Plan

## Branches

**Base**: `feature/closet-two-level-tabs`

このフィーチャは状態 / サービス / View が密結合しており、単一のベースブランチで実装する。全タスクはベースブランチに直接コミットする。

## Tasks

### Branch: `feature/closet-two-level-tabs`

- [x] 1. クローゼットタブ状態の SSOT を State 層に構築する
  - [x] 1.1 大タブ列挙と「大タブ ↔ 小タブ集合」マッピングを Home.State 配下で定義する
    - 大タブを「体」「服」の 2 値だけ保持する列挙を新設する
    - 「体 = Body, Face, Tail, FaceMakeup」「服 = Cloth, HandAccessory, HeadAccessory, LegAccessory, Effect」を表す静的読取専用マップを 1 箇所だけ定義する (`FaceMakeup` は要件 2.5 の確定どおり「体」配下)
    - 大タブから既定の小タブ (体 → Face / 服 → Cloth) を引き当てる参照を同マップ内で持つ
  - [x] 1.2 状態保持と変更通知を担うクラスを実装する
    - 現在の大タブと小タブを保持し、初期値を「体 + Face」とする
    - 大タブ・小タブそれぞれの変更を通知するイベントを差分通知 (値が変化したときのみ発火) で実装する
    - 大タブから小タブ集合・既定値、小タブから大タブを引く参照メソッドを公開する
  - [x] 1.3 マッピングの不変条件を初期化時に検証する
    - 全 `OutfitType` がいずれか 1 つの大タブにのみ属することを起動時に走査・検証し、違反時はクラス名コンテキスト付きでエラーログを出す
    - 重複や未割当が検知されてもクラッシュさせず、既定状態 (体 + Face) で動作継続する
  - _Requirements: 1.4, 1.5, 2.1, 2.2, 2.3, 2.4, 2.5, 8.2_

- [x] 2. クローゼットタブ遷移ロジックを Service 層に実装する
  - [x] 2.1 既定状態リセット操作を実装する
    - 大タブ「体」と小タブ `Face` をまとめて再設定する操作を公開する
    - クローゼットを再 Open するたびに常に既定状態へ戻す入口とする
  - [x] 2.2 大タブ選択操作を実装する
    - 同じ大タブが再選択された場合は状態書き込みもイベント発火も行わない (no-op)
    - 異なる大タブが選択されたら大タブを更新し、続けて小タブを既定値 (体 → Face / 服 → Cloth) に同期する
    - 大タブ切替時に小タブが変わる経路でも、状態書き込みは必ずタブ状態クラス経由で行う
  - [x] 2.3 小タブ選択操作を実装する
    - 同じ小タブが再選択された場合は no-op で再構築を抑止する
    - 現在の大タブのマッピングに含まれない小タブ値が来た場合はクラス名コンテキスト付きでエラーログを出し、状態は変更しない
  - _Requirements: 1.4, 1.5, 3.1, 3.2, 3.4, 4.1, 4.2, 4.3, 5.4, 8.3_

- [x] 3. グリッドサービスをタブフィルタと OnOpen 連鎖に対応させる
  - [x] 3.1 タブ状態とタブサービスを依存に追加する
    - グリッドサービスのコンストラクタで現在の小タブとタブ操作の双方を解決できるようにする
    - 既存メソッド (Initialize / LoadData / OnCellViewSelected) のシグネチャは変更しない
  - [x] 3.2 OnOpen ハンドラで「リセット → 初期化 → 描画」を 1 連の処理に統合する
    - クローゼットの OnOpen を契機に、まずタブを既定状態にリセットしてから既存の初期化と LoadData を 1 度だけ呼ぶ
    - リセット中に発生する小タブ変更通知を抑止する内部フラグを導入し、二重描画が起きないようにする
    - 抑止フラグは try/finally で必ず解除し、想定外の例外で抑止が残らないようにする
  - [x] 3.3 小タブ変更通知の購読と防御ガードを実装する
    - 起動時に小タブ変更通知を 1 度だけ購読し、別寿命管理を持たない (サービスとタブ状態は同じスコープ寿命)
    - リセット中フラグが立っているとき、およびクローゼット UI が非表示のときの通知は no-op にする
    - クローゼット表示中の通知ではフィルタ済み LoadData を呼び、スクロール位置を先頭へリセットする
  - [x] 3.4 LoadData にフィルタ条件を差し込み既存の責務を維持する
    - マスター走査ループで現在の小タブと一致しない `OutfitType` をスキップし、結果リストにフィルタ済みの装備のみ積む
    - フィルタ後の件数が 0 件でも壊れず空グリッドとして成立させる
    - フィルタ後でも装備中の装備が表示対象に含まれる場合は選択枠を点灯させる従来挙動を維持する
    - フィルタ範囲外の他 `OutfitType` の装備状態を一切書き換えない
    - LoadData 終了時にスクロール位置を先頭にリセットする
  - _Requirements: 3.3, 3.4, 4.4, 5.1, 5.2, 5.3, 6.1, 6.2, 6.3, 6.4, 8.4_

- [x] 4. (P) タブ要素単体の見た目とクリック発火を担う presenter View を実装する
  - [x] 4.1 (P) 大タブ要素の選択中・非選択中の見た目切替を実装する
    - 既存の `HeadingItem.prefab` に追加する想定で、選択中・非選択中の背景スプライトを Inspector 経由で受け取る
    - 親コンテナから受け取ったクリック時コールバックをボタンに結線し、自身は外部依存を持たない
  - [x] 4.2 (P) 小タブ要素の選択中・非選択中の見た目切替を実装する
    - 既存の `TabItem.prefab` に追加する想定で、選択中・非選択中の背景スプライトとアイコン色 (白 / pink) を Inspector 経由で受け取る
    - 親コンテナから受け取ったクリック時コールバックをボタンに結線する
  - _Requirements: 7.1, 7.2, 7.3, 7.4, 7.5_

- [x] 5. タブ群コンテナ View を実装する
  - [x] 5.1 大タブコンテナ View を実装する
    - 静的に「体」「服」の 2 つのタブ要素を Inspector 参照で保持する
    - 各要素のクリック時に対応する大タブ値でタブサービスの選択操作を呼ぶ
    - タブ状態の大タブ変更通知を購読し、各要素の選択中スプライトを切り替える
    - 自身の破棄時にイベント購読を確実に解除する
  - [x] 5.2 小タブコンテナ View を実装する
    - 大タブ変更通知を購読し、現在の大タブに対応する小タブ集合を取得して動的に要素を再構築する
    - 既存子要素は破棄または再利用し、生成数は集合サイズと一致させる
    - 各要素にアイコン (`Assets/UI/Home/Closet/Textures/Icons/` 既存素材のみ) を Inspector マップから引き当ててバインドする
    - 未登録の `OutfitType` がアイコンマップに存在する場合はクラス名コンテキスト付きでエラーログを出し、空アイコンでフォールバック表示する
    - 各要素のクリック時にタブサービスの小タブ選択操作を呼ぶ
    - 小タブ変更通知を購読し、選択中要素の見た目を切り替える
    - 自身の破棄時にイベント購読を解除する
  - _Requirements: 1.1, 1.2, 1.3, 1.4, 1.5, 2.3, 7.1, 7.2, 7.3, 7.4, 7.5_

- [x] 6. クローゼット ルート View をタブ View ホスト化する
  - [x] 6.1 大・小タブコンテナ View への参照を Inspector 公開する
    - クローゼット ルート View に大タブ / 小タブ各コンテナへの参照フィールドを追加する
    - 既存の戻るボタン・スクローラ参照は維持し、注入される DI 依存も既存通り壊さない
  - [x] 6.2 子タブ View にタブ状態とタブサービスをバインドする
    - クローゼット ルート View の初期化時にタブサービスとタブ状態を受け取る
    - 受け取った参照をそのまま大・小タブコンテナ View に渡してバインドする
    - リセット連鎖の責務はグリッドサービス側に集約しているため、ルート View 側で `OnOpen` リスナーを直接登録しない
  - _Requirements: 1.1, 8.1, 8.5_

- [ ] 7. DI 登録とプレハブ・シーン配線を統合する
  - [x] 7.1 HomeScope に新規 State / Service を登録する
    - タブ状態とタブサービスを既存スコープと同じ寿命で登録する (具象登録のみ、インターフェース抽象は導入しない)
    - グリッドサービスのコンストラクタが追加依存を解決できることを確認する
  - [ ] 7.2 既存プレハブにタブ要素 presenter を組み込む _(Unity Editor 作業)_
    - `HeadingItem.prefab` に大タブ要素 presenter を追加し、選択中・非選択中スプライトをアサインする
    - `TabItem.prefab` に小タブ要素 presenter を追加し、選択中・非選択中背景スプライトとアイコン色 (`Assets/UI/Home/Closet/Textures/` および `Icons/` 配下の既存素材のみ) をアサインする
    - 新規アセットを 1 つも追加しない
  - [ ] 7.3 Home シーンのクローゼット UI 階層にタブコンテナを配置する _(Unity Editor 作業)_

    - 大タブコンテナの「体」「服」要素、および小タブコンテナのアイコンマップ (`OutfitType` → `Sprite`) を結線する
    - クローゼットを Open した直後にグリッドが「Face」フィルタで描画され、大タブ「服」を選択すると `Cloth` に切り替わる挙動を Editor 上で確認する
  - _Requirements: 1.1, 1.2, 1.3, 7.5, 8.3, 8.5_
