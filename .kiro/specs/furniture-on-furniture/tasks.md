# Implementation Plan

## Branches

**Base**: `feature/furniture-on-furniture`

| Branch | Tasks | Goal |
|--------|-------|------|
| `feature/furniture-on-furniture-foundation` | 1-2 | 座標変換の共通化基盤が完成し、既存機能が正常動作する |
| `feature/furniture-on-furniture-placement` | 3-4 | 家具の上に家具を配置できる機能が完成する |

## Tasks

### Branch: `feature/furniture-on-furniture-foundation`

- [x] 1. 座標変換サービスの作成
  - [x] 1.1 アイソメトリック座標変換の共通サービスを実装する
    - cellSizeとangleから座標変換用の軸ベクトルを計算する機能を実装
    - グリッド座標からオフセット座標への変換機能を実装
    - オフセット座標からグリッド座標への変換機能を実装（丸め処理含む）
    - 状態を持たない純粋な計算クラスとして設計
  - _Requirements: -_

- [x] 2. グリッドサービスのリファクタリング
  - [x] 2.1 既存のグリッドサービスを座標変換サービスを使用するよう変更する
    - コンストラクタで座標変換サービスのインスタンスを生成
    - 床グリッド座標からワールド座標への変換処理を座標変換サービスに委譲
    - ワールド座標から床グリッド座標への変換処理を座標変換サービスに委譲
    - 既存の動作に影響がないことを確認
  - _Requirements: -_

### Branch: `feature/furniture-on-furniture-placement`

- [x] 3. 家具上面の配置グリッドコンポーネントを実装する
  - [x] 3.1 配置可能グリッドの基本構造を実装する
    - 家具オブジェクトの子としてアタッチ可能なMonoBehaviourを作成
    - 2D Colliderを必須コンポーネントとして設定
    - グリッドサイズをインスペクターから設定可能にする
    - グリッド設定（cellSize, angle）をIsoGridSettingsViewから取得
  - [x] 3.2 座標変換機能を実装する
    - ローカルグリッド座標からワールド座標への変換機能を実装
    - ワールド座標からローカルグリッド座標への変換機能を実装
    - グリッド座標の有効範囲チェック機能を実装
  - [x] 3.3 配置状態管理機能を実装する
    - グリッドセル配列による占有状態の管理機能を実装
    - オブジェクト配置時のセル占有登録機能を実装
    - オブジェクト削除時のセル解放機能を実装
    - 配置されたオブジェクトの位置追跡機能を実装
  - [x] 3.4 配置制約の判定機能を実装する
    - 壁配置用家具（IsWallPlacement=true）の配置を拒否する機能を実装
    - フットプリントがグリッドサイズを超える場合の配置拒否機能を実装
    - セルが既に占有されている場合の配置拒否機能を実装
    - 複数オブジェクトの同時配置をサポート
  - _Requirements: 1.1, 1.2, 1.3, 1.4, 3.1, 3.2, 3.3_

- [x] 4. ドラッグ操作での家具上配置を実装する
  - [x] 4.1 ドラッグ開始時の解除処理を実装する
    - ドラッグ開始時に現在のオブジェクトが配置グリッド上にあるか判定
    - 配置グリッド上にある場合はオブジェクトを解除
    - 元の配置グリッドと位置を保存
  - [x] 4.2 ドラッグ終了時の配置判定を実装する
    - RayCastで配置グリッドとの衝突をチェック
    - 自身のColliderに属する配置グリッドは検出対象から除外
    - 衝突した配置グリッドへの配置可否を判定
  - [x] 4.3 配置処理とフォールバックを実装する
    - 配置可能な場合は配置グリッドにオブジェクトを配置
    - 配置位置をグリッドにスナップ
    - 配置不可または衝突なしの場合は通常の床配置処理を実行
    - 配置拒否時は元の位置（配置グリッドまたは床）に戻す
  - _Requirements: 2.1, 2.2, 2.3, 3.4_
