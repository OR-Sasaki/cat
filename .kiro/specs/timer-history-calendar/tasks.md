# Implementation Plan

## Branches

**Base**: `feature/timer-history-calendar`

| Branch | Tasks | Goal |
|--------|-------|------|
| `feature/timer-history-calendar-timer-record` | 1-5 | 集中時間がタイマー利用 (正常完了 / 中断 / Pause/Quit) で日別に永続化される。Pomodoro が Flush を呼び、PlayerPrefs に秒単位で蓄積される動作確認可能な状態 |
| `feature/timer-history-calendar-history-calendar` | 6-11 | History シーン単独でカレンダー UI / 連続日数 / 選択日 / 月合計 / 当日合計が表示できる。直接 History.unity を再生して 4 段階アイコン差し替えと月送り・選択操作が成立 |
| `feature/timer-history-calendar-home-link` | 12-13 | Home フッターの History ボタンから履歴画面へ遷移でき、タイマー → 履歴の往復で当日記録が反映される統合状態 |

- [x] 2. 集中時間記録の単一ソースとなるサービスと状態を実装する
  - [x] 2.1 日付→秒数の保持を担う状態オブジェクトを作る
    - 内部 Dictionary を保持し、外部にはイミュータブル写像のみを返す API を提供する

  - [x] 2.2 集中時間記録サービスの参照系 API を実装する
    - 指定日の秒数取得、指定月の合計秒数取得、当日合計秒数取得、全記録のイミュータブル写像取得を提供する

  - [x] 2.3 集中時間記録サービスの加算 API と永続化を実装する
    - 加算 API は呼出時点のローカル日付に紐付け、0 以下の入力は無視する
    - 加算後に同期書込でスナップショットを保存し、参照値の整合を保つ
    - 加算完了イベント (日付 / 加算秒数 / 当日合計) を発火する
  - [x] 2.4 サービスのロード処理と初期化フローを実装する

  - [x] 2.5 ルートスコープに集中時間記録サービス／状態を Singleton 登録する
